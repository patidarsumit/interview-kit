## Project Architecture Answer

```text
At a high level, we followed a modular Angular architecture.

Business features were separated into feature modules or feature areas. Shared components, pipes, directives, and reusable utilities were kept separate so they could be used across multiple screens.

Core application concerns like authentication, route guards, HTTP interceptors, configuration, error handling, and logging were handled centrally.

For API communication, we used Angular services. Components were kept focused on UI and interaction logic, while services handled HTTP calls and reusable business operations.

For complex forms, we used reactive forms with nested form groups, dynamic validations, and conditional fields. For async flows, we used RxJS operators to manage API calls, loading states, error handling, and subscriptions.

The main goal of the architecture was maintainability: new features should be easy to add, components should be reusable, and the code should be testable.
```

## Biggest Technical Challenge

Use this answer if they ask about performance.

```text

Some screens were becoming slow because of heavy data rendering, repeated API calls, and expensive UI updates. The user experience was impacted, especially on pages with complex forms and large datasets.

I analyzed the issue using browser DevTools, network calls, component rendering behavior, and bundle size. Then I worked on lazy loading, OnPush change detection, virtual scrolling where large lists were involved, and reducing unnecessary calculations in templates.

We also optimized API integration and improved how loading and error states were handled.

As a result, application performance improved significantly, and in one key area we improved the Lighthouse score from 58 to 87. This also made the UI smoother and easier to maintain.
```
