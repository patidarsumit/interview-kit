# Citi Karat Interview Prep - Angular / Frontend Developer

Assumption: "city bank" means Citi / Citibank.

This folder is prepared for a Karat technical screen for an Angular / frontend developer role at Citi.

## Expected Interview Shape

Based on Karat's own candidate resources and public candidate reports, expect a 45-60 minute live video interview. The common format is:

1. Brief introduction.
2. Role-specific discussion / knowledge questions.
3. Live coding in Karat Studio or a similar browser-based coding environment.
4. Complexity, edge-case, debugging, and explanation follow-ups.

Your understanding is directionally correct:

```text
Around 20 minutes: experience + frontend / Angular knowledge discussion
Around 40 minutes: live coding
```

Karat's official candidate page describes a 60-minute interview with intro, discussion questions, and about 40 minutes of programming. A Karat support article describes 10-25 minutes of role-specific discussion or code review and 30-45 minutes of coding. So prepare for either split.

## Highest Probability Areas

- JavaScript / TypeScript fundamentals.
- Angular components, lifecycle, routing, guards, forms, services, HTTP interceptors.
- RxJS operators and async data flow.
- State management with RxJS / NgRx.
- REST API integration and microservice frontend patterns.
- HTML, CSS, responsive layout, accessibility.
- Small DSA problems in JavaScript / TypeScript.
- Debugging or extending existing code.
- Time and space complexity explanation.
- Clear communication while coding.

## Your Strong Positioning

```text
I am an Angular-focused frontend engineer who can build enterprise web applications end to end: reusable components, reactive forms, routing, API integration, authentication, state management, performance, accessibility, testing, and production debugging.

For Citi, I understand the frontend is not just UI. It must be reliable, secure, accessible, maintainable, and integrated cleanly with backend microservices in a regulated financial environment.
```
